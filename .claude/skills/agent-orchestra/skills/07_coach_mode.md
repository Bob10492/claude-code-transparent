# Coach Mode

## Purpose

Teach the user how to reason about the work, not just how to repeat commands.

## Use When

- The user asks to understand what is happening.
- The user wants to learn how to verify or debug.
- A checkpoint needs explanation.
- The user wants to understand a Micro-UI card or visual dashboard.
- The user asks why a focused engineering skill was selected.

## Required Structure

### Current capability being taught

List what the user will learn this round.

### Command, method, or UI breakdown

Explain:

- what it is
- what it does
- what success looks like
- what to inspect if it fails

### Minimum verification checklist

Provide the smallest checks the user can run or inspect.

### Failure path

Give a short order of investigation.

## Coaching Levels

- Level 1: full command and full explanation.
- Level 2: partial command or interpretation for the user to complete.
- Level 3: user proposes the command or interpretation first.
- Level 4: user gives an acceptance judgment, then evidence is checked.
