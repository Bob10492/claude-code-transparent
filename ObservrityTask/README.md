# ObservrityTask 目录索引

当前目录按“资料输入 / 系统版本”分层，便于后续继续并列扩展 `v2`、`v3`。

## 目录结构

- `00-资料输入`
  - 原始任务书、输入说明、参考 PDF / 图片
- `10-系统版本`
  - `v1/`
    - `01-总览`
    - `02-Schema与指标`
    - `03-样例`
    - `04-专题研究`
  - `v2/`
    - `01-总览`
    - `02-实施任务书`
    - `03-数据模型`
    - `04-Scenario集`
    - `05-Variant与实验`
    - `06-运行报告`

## 阅读顺序

1. 先看 `10-系统版本/v1/01-总览/当前可观测系统V1深度研究报告.md`
2. 再看 `10-系统版本/v1/02-Schema与指标/`
3. 需要实操时看 `10-系统版本/v1/03-样例/`
4. 需要回溯输入背景时看 `00-资料输入/`
5. 需要开始规划下一代平台时看 `10-系统版本/v2/01-总览/可观测系统V2北极星与评测模型草案.md`
6. 需要直接推进 V2 开工时看 `10-系统版本/v2/02-实施任务书/可观测系统V2第一阶段实施任务书.md`
7. 需要把 V2 第一阶段拆成可执行步骤时看 `10-系统版本/v2/02-实施任务书/可观测系统V2第一阶段执行清单.md`
8. 需要落地 V2 数据模型和工作骨架时看 `10-系统版本/v2/03-数据模型/`、`10-系统版本/v2/04-Scenario集/`、`10-系统版本/v2/05-Variant与实验/`
9. 需要查看 V2 run 证据报告时看 `10-系统版本/v2/06-运行报告/`
